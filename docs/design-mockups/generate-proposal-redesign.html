Section 02
2-Phase Modal Flow
Clicking the button opens a modal. Phase 1 uses Sonnet + chat context to author a content preview. Phase 2 collects cover fields, then assembles the branded DOCX.
Step 1
Chat in RFx Responder
User analyses requirements, maps to iMocha capabilities. Chat history builds context.
Step 2
Click "Generate Proposal"
Modal opens. Sonnet call reads chat summary + bid data → authors exec summary, scope intro, deliverables.
Step 3
Review content preview
User reads AI-authored sections. Can regenerate if needed.
Step 4
Fill cover fields + confirm
prepared_for, spoc_name, spoc_email. User hits "Generate DOCX".
Done
Branded DOCX
Downloaded to browser + saved to Knowledge Hub under this bid.
