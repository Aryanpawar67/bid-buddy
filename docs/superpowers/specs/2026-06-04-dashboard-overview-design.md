# Dashboard Overview — Design Spec
_Date: 2026-06-04_

## Goal

Replace the current `/dashboard` pipeline split-pane with a command-centre overview for Pre-Sales / Bid Managers. The pipeline split-pane moves to `/pipeline`. The new dashboard gives a 3-second health read (KPI strip) plus a prioritised action list of bids that need attention.

## 2. KPI Cards

Five stat cards in a single row. All values derived from the existing `useBids()` query via a single `useMemo` — no additional Supabase queries.

| Card | Metric | Computation |
|---|---|---|
| Active bids | count | `bids.filter(b => b.status === "active").length` |
| Pipeline value | formatted sum | `fmtMoney(sum of value where status === "active")` |
| Expiring soon | count | active bids with deadline ≤ 5 days (uses `urgencyClass` threshold) |
| Overdue | count | active bids with deadline already passed; card text is `text-destructive` when > 0 |
| Avg Go/No-Go | mean score /100 | mean of `gonogo_score` where not null; renders `—` if no bids scored yet |

Card styling follows the existing `Stat` component pattern from `queue.tsx`:
```
bg-card hairline border rounded-lg p-3
text-[10px] uppercase tracking-wider text-muted-foreground   ← label
text-[20px] font-medium leading-none mt-1                    ← value
```

## 3. Needs Attention List

A prioritised list of bids that require action, rendered below the KPI strip.

### Inclusion criteria
Only `status === "active"` bids are considered. A bid is included if **any** of the following are true:
- Deadline is overdue or ≤ 5 days away
- `priority === "high"`
- `gonogo_decision === null` AND `stage === "deal_qualification"` (unscored at qualification)

### Sort order
1. Overdue (deadline < today) — ascending by deadline
2. Expiring ≤ 5 days — ascending by deadline
3. High priority (not yet deadline-urgent)
4. Unscored deal qualification

### Row anatomy
```
[Client name · Title]  [Stage badge]  [Priority dot]  [Deadline]  [Go/No-Go chip?]  [Open →]
```

- **Stage badge:** `stageLabel(b.stage)` — `bg-primary-soft text-primary text-[10px] px-2 py-0.5 rounded-sm`
- **Priority dot:** high = `bg-accent` (orange), medium = `bg-warning`, low = `bg-muted-foreground/30`
- **Deadline:** `urgencyClass(b.deadline)` for colour + label text
- **Go/No-Go chip:** only rendered if `gonogo_decision` is set
  - `"go"` → `bg-success-soft text-success-foreground`
  - `"no_go"` → `bg-danger-soft text-danger-foreground`
  - `"conditional_go"` → `bg-warning-soft text-warning-foreground`
- **Open →** Link to `/bids/$id`

### Empty state
When no bids qualify: a green-tinted card reading "All clear — no urgent bids. Pipeline looks healthy."

## 4. All Active Bids (Collapsible)

Below the attention list, a collapsible section ("All active bids · N") showing every active bid in the same row format, sorted by deadline ascending. Collapsed by default. Uses the existing `Collapsible` Radix component already in the project.

---

## 5. Data

No new queries. Everything derives from `useBids()` which already fetches all bid fields ordered by deadline.

```ts
const { data: bids = [], isLoading } = useBids();
const stats = useMemo(() => computeStats(bids), [bids]);
const attention = useMemo(() => computeAttention(bids), [bids]);
const allActive = useMemo(() => bids.filter(b => b.status === "active"), [bids]);
```
